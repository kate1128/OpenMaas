## 2. API 接入

### 2.1 认证（JWT 特殊鉴权）

智谱 AI 使用自定义 JWT Token，需要用 **API Key** 中的 `id.secret` 拆分并签名：

```
API Key 格式：{id}.{secret}
例如：abc123.mysecretkey456
```

```python
import jwt
import time

def generate_zhipu_token(api_key: str) -> str:
    api_key_id, api_key_secret = api_key.split(".")
    payload = {
        "api_key": api_key_id,
        "exp": int(time.time()) + 3600,  # 1小时有效期
        "timestamp": int(time.time())
    }
    return jwt.encode(
        payload,
        api_key_secret,
        algorithm="HS256",
        headers={"alg": "HS256", "sign_type": "SIGN"}
    )

# 使用时
token = generate_zhipu_token("abc123.mysecretkey456")
headers = {"Authorization": f"Bearer {token}"}
```

> **注意：** 这是智谱 AI 最特殊的地方。每次调用需要实时生成 JWT Token，官方 SDK 内置了这个逻辑。

## 8. MaaS 接入评估

### 8.1 Adapter 复杂度：低（JWT 需封装）

核心挑战是 JWT 鉴权，需要在 Adapter 层实现 Token 自动刷新：

```python
class ZhipuAdapter:
    def __init__(self, api_key: str):
        self.api_key = api_key
        self._token_cache = None
        self._token_expire = 0
    
    def get_auth_header(self) -> dict:
        now = time.time()
        if self._token_cache is None or now >= self._token_expire - 60:
            self._token_cache = self._generate_token()
            self._token_expire = now + 3600
        return {"Authorization": f"Bearer {self._token_cache}"}
    
    def _generate_token(self) -> str:
        kid, secret = self.api_key.split(".")
        return jwt.encode(
            {"api_key": kid, "exp": int(time.time()) + 3600, "timestamp": int(time.time())},
            secret, algorithm="HS256",
            headers={"alg": "HS256", "sign_type": "SIGN"}
        )
```

### 8.2 接入配置

```yaml
vendor: zhipu
base_url: "https://open.bigmodel.cn/api/paas/v4"
auth_type: jwt_custom              # 自定义 JWT
api_key_env: ZHIPU_API_KEY         # 格式: id.secret
adapter: zhipu_v4
token_refresh_interval: 3600s     # JWT 有效期 1 小时
timeout: 60s
```

### 8.3 注意事项

1. **JWT 自动刷新**：Token 1小时过期，必须在 Adapter 层实现自动刷新，不能复用过期 Token
2. **免费模型限额**：glm-4-flash 有每天 1000 次上限，超出后自动降级为付费，需监控
3. **模型定价对称**：输入和输出同价（与 OpenAI 输出 > 输入的定价模式不同），成本估算较简单
4. **图片计费**：视觉模型图片部分按像素/分辨率额外计费
5. **cogview 图片生成**：按张计费，非 Token 计费，需在 MaaS 计费模块中特殊处理
