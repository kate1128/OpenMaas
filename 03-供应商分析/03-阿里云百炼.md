## 2. API 接入

### 2.1 认证

```http
Authorization: Bearer sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
Content-Type: application/json
```

API Key 在 **阿里云百炼控制台** → API Key 管理中创建。

### 2.2 Base URL（OpenAI 兼容端点）

```
https://dashscope.aliyuncs.com/compatible-mode/v1
```

> 百炼提供两套 API：  
> - **OpenAI 兼容端点**（推荐接入）：`/compatible-mode/v1/chat/completions`  
> - **DashScope 原生 API**：格式不同，不推荐用于 MaaS 接入

### 2.3 SDK

推荐使用 OpenAI SDK，仅修改 `base_url`：

```python
from openai import OpenAI

client = OpenAI(
    api_key="sk-xxx",
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1"
)

response = client.chat.completions.create(
    model="qwen-max",
    messages=[{"role": "user", "content": "你好"}]
)
```

## 3. 调用格式

### 3.1 标准请求（与 OpenAI 完全兼容）

```json
{
  "model": "qwen-max",
  "messages": [
    {"role": "system", "content": "你是一个有帮助的助手"},
    {"role": "user",   "content": "帮我写一首关于秋天的诗"}
  ],
  "temperature": 0.7,
  "max_tokens": 2048,
  "stream": false
}
```

### 3.2 扩展参数（Qwen 特有）

```json
{
  "model": "qwen-max",
  "messages": [...],
  "extra_body": {
    "enable_search": true,        // 启用联网搜索（部分模型支持）
    "search_options": {
      "forced_search": false       // 是否强制搜索（不判断是否需要）
    }
  }
}
```

### 3.3 Vision（多模态）

```json
{
  "model": "qwen-vl-max",
  "messages": [
    {
      "role": "user",
      "content": [
        {
          "type": "image_url",
          "image_url": {"url": "https://example.com/image.jpg"}
        },
        {"type": "text", "text": "图片里有什么？"}
      ]
    }
  ]
}
```

### 3.4 Embedding

```json
// 请求
{
  "model": "text-embedding-v3",
  "input": ["Hello world", "你好世界"],
  "dimensions": 1024,
  "encoding_format": "float"
}
```

## 8. MaaS 接入评估

### 8.1 接入方案

```yaml
vendor: aliyun_bailian
base_url: "https://dashscope.aliyuncs.com/compatible-mode/v1"
auth_type: bearer_token
api_key_env: DASHSCOPE_API_KEY
adapter: openai_compatible        # 使用 OpenAI 兼容 Adapter
timeout: 60s
```

### 8.2 模型 ID 映射建议

```yaml
# MaaS 统一模型名 → 百炼模型 ID
model_mapping:
  "qwen-max":    "qwen-max-latest"
  "qwen-plus":   "qwen-plus-latest"
  "qwen-turbo":  "qwen-turbo-latest"
  "qwen-vl-max": "qwen-vl-max"
```

### 8.3 注意事项

1. **OpenAI 兼容端点成熟稳定**，直接用 OpenAI SDK 即可，无需自定义 Adapter
2. **联网搜索需要 extra_body**，需在 MaaS Adapter 层支持传递扩展参数
3. **超长上下文限制**：`qwen-long` 的 10M 上下文极大，但 Token 计费也相应增加，需在路由策略中谨慎使用
4. **图片计费规则**：视觉模型按图片分辨率额外计费（非按 token），需单独核算
5. **模型版本策略**：建议使用带 `-latest` 后缀的模型名享受自动更新；生产环境如需稳定性，使用带日期的版本
6. **企业认证**：企业账号可享受更高配额和更优惠的合同价

## 9. 常见错误码

| 错误码 | 含义 | 处理建议 |
|:------:|-----|---------|
| InvalidApiKey | API Key 无效 | 检查 Key 是否正确 |
| Arrearage | 账户欠费 | 充值阿里云账户 |
| InvalidModel | 模型不存在或无权限 | 检查模型 ID |
| RequestThrottled | 超出速率限制 | 退避重试 |
| ContextLengthExceeded | 超出上下文长度 | 截断 prompt 或换 qwen-long |
