## 2. API 接入

### 2.1 认证

```http
Authorization: Bearer ark-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
Content-Type: application/json
```

API Key（ARK Key）在**火山方舟控制台** → API Key 管理中创建，前缀为 `ark-`。

### 2.2 Base URL

```
https://ark.cn-beijing.volces.com/api/v3
```

**Chat Completions：**
```
POST https://ark.cn-beijing.volces.com/api/v3/chat/completions
```

### 2.3 重要特殊性：Endpoint ID

> **豆包最大的特殊性**：`model` 字段需要填写**接入点 ID**（Endpoint ID），而不是通用模型名。
>
> 接入点需要在**火山方舟控制台** → 模型接入 → 创建接入点后获得。
>
> 格式：`ep-20240611112233-xxxxx`（每个账号不同）

### 2.4 SDK

```bash
pip install volcengine-python-sdk[ark]   # 官方 SDK
# 或直接使用 OpenAI SDK
pip install openai
```

```python
from openai import OpenAI

client = OpenAI(
    api_key="ark-xxx",
    base_url="https://ark.cn-beijing.volces.com/api/v3"
)

response = client.chat.completions.create(
    model="ep-20240611112233-xxxxx",   # 使用 Endpoint ID！
    messages=[{"role": "user", "content": "你好"}]
)
```

## 3. 调用格式

### 3.1 标准 Chat 请求

```json
{
  "model": "ep-20240611112233-xxxxx",
  "messages": [
    {"role": "system", "content": "你是豆包，一个有帮助的 AI 助手"},
    {"role": "user",   "content": "介绍一下火山引擎"}
  ],
  "temperature": 0.7,
  "max_tokens": 4096,
  "stream": false
}
```

> 除 `model` 字段使用 Endpoint ID 外，其余字段与 OpenAI 完全相同。

### 3.2 Vision（多模态）

```json
{
  "model": "ep-vision-xxxxx",
  "messages": [
    {
      "role": "user",
      "content": [
        {
          "type": "image_url",
          "image_url": {"url": "https://example.com/image.jpg"}
        },
        {"type": "text", "text": "这张图片描述了什么？"}
      ]
    }
  ]
}
```

### 3.3 Function Calling

```json
{
  "model": "ep-20240611112233-xxxxx",
  "messages": [{"role": "user", "content": "当前北京时间"}],
  "tools": [
    {
      "type": "function",
      "function": {
        "name": "get_current_time",
        "description": "获取指定城市当前时间",
        "parameters": {
          "type": "object",
          "properties": {
            "timezone": {"type": "string", "description": "时区，如 Asia/Shanghai"}
          }
        }
      }
    }
  ]
}
```

### 3.4 Embedding

```json
{
  "model": "ep-embedding-xxxxx",

```

## 8. MaaS 接入评估

### 8.1 Adapter 复杂度：低（Endpoint ID 映射）

核心挑战：`model` 字段需要 Endpoint ID，MaaS 层需要维护模型名 → Endpoint ID 的映射表。

```yaml
vendor: doubao
base_url: "https://ark.cn-beijing.volces.com/api/v3"
auth_type: bearer_token
api_key_env: ARK_API_KEY
adapter: openai_compatible

# 必须维护模型名 → Endpoint ID 的映射
endpoint_mapping:
  "doubao-pro-32k":     "ep-20240611112233-aaaaa"
  "doubao-lite-32k":    "ep-20240611112234-bbbbb"
  "doubao-pro-128k":    "ep-20240611112235-ccccc"
  "doubao-embedding":   "ep-20240611112236-ddddd"
```

```python
class DoubaoAdapter:
    def __init__(self, endpoint_mapping: dict):
        self.endpoint_mapping = endpoint_mapping
    
    def transform_request(self, req: dict) -> dict:
        model_name = req.get("model", "")
        ep_id = self.endpoint_mapping.get(model_name, model_name)
        req["model"] = ep_id
        return req
```

### 8.2 注意事项

1. **Endpoint ID 必须在控制台手动创建**，且每个账号的 ID 不同，不可跨账号复用
2. **每个模型需要单独创建 Endpoint**，批量接入时管理成本较高
3. **Endpoint 存在失效风险**：如果删除或暂停接入点，对应的 model ID 就失效，需要更新映射
4. **角色扮演模型**适合 C 端对话产品，不适合企业数据处理
5. **北京区域**：目前主要在 `cn-beijing` 区域，上海/杭州等区域可能支持有限

## 9. 常见错误码

| 状态码 | 含义 | 处理建议 |
|:------:|-----|---------|
| 400 | 请求格式错误 | 检查 Endpoint ID 是否有效 |
| 401 | ARK Key 无效 | 检查 API Key |
| 404 | Endpoint 不存在 | 检查控制台 Endpoint 配置 |
| 429 | 速率超限 | 退避重试 |
| 500 | 服务错误 | 重试 |
