## 一、中间件全景图

```
┌─────────────────────────────────────────────────────────────────────┐
│                        MaaS 平台中间件栈                              │
├───────────────┬──────────────────────────────────────────────────────┤
│  接入层        │  Nginx（反向代理 / TLS 终止 / 限流）                  │
│               │  Istio Service Mesh（服务间流量管理 / mTLS / 灰度）   │
├───────────────┼──────────────────────────────────────────────────────┤
│  数据存储层    │  PostgreSQL（关系型主库）                             │
│               │  Redis（缓存 / 限流 / 会话 / 向量检索）               │
│               │  ClickHouse（OLAP Trace 分析）                        │
│               │  Qdrant（语义向量检索）                                │
│               │  MinIO / S3（对象存储）                                │
├───────────────┼──────────────────────────────────────────────────────┤
│  消息队列层    │  Kafka（事件流 / 异步解耦 / 审计日志投递）             │
├───────────────┼──────────────────────────────────────────────────────┤
│  可观测性层    │  Prometheus（指标采集）                                │
│               │  Grafana（Dashboard 可视化）                           │
│               │  OpenTelemetry Collector（Trace 采集）                 │
│               │  Jaeger（分布式 Trace 查询 UI）                        │
│               │  Vector（日志聚合 / 转换 / 路由）                      │
├───────────────┼──────────────────────────────────────────────────────┤
│  安全层        │  HashiCorp Vault（密钥管理 / 动态凭证）                │
└───────────────┴──────────────────────────────────────────────────────┘
```

## 二、选型对比矩阵

| 中间件 | 版本要求 | 角色 | 核心理由 | 替代方案 | 为何不选替代方案 |
|-------|---------|------|---------|---------|----------------|
| PostgreSQL | ≥ 15 | 关系型主库 | JSONB + 行级安全（RLS）+ 分区表 | MySQL | MySQL RLS 弱、JSONB 性能差 |
| Redis | ≥ 7.2 | 缓存 / 限流 / 向量 | Redis Stack 内置 RedisSearch 向量模块，Cluster 模式横向扩展 | Memcached | 无持久化、无数据结构多样性 |
| Kafka | ≥ 3.6（KRaft） | 事件流 | 高吞吐、持久化、可回放，KRaft 去 ZooKeeper 依赖 | RabbitMQ / Pulsar | RabbitMQ 吞吐低；Pulsar 运维复杂 |
| ClickHouse | ≥ 24.3 | OLAP 分析 | 列存压缩比高、亿级 Trace 查询毫秒级响应 | Elasticsearch | ES 写入放大严重、存储成本高 |
| Qdrant | ≥ 1.9 | 语义向量检索 | 纯向量数据库、HNSW 索引、gRPC API，Rust 实现低延迟 | Redis Vector | Redis Vector 不支持复杂过滤 + 向量混合检索 |
| MinIO | ≥ RELEASE.2024 | 对象存储 | S3 兼容 API、私有化部署、纠删码 | 阿里云 OSS / AWS S3 | 私有部署场景需离线；公有云可直接替换 |
| Kafka（兼 MinIO） | — | 审计归档 | 冷数据从 Kafka 导出到 MinIO Parquet | — | — |
| Prometheus | ≥ 2.50 | 指标采集 | 业界标准、PromQL 表达力强、服务原生暴露 /metrics | VictoriaMetrics | VM 生产验证少；Prometheus 生态更完善 |
| Grafana | ≥ 10.4 | 可视化 | 与 Prometheus / ClickHouse / Loki 多数据源统一 | Kibana | Kibana 强依赖 ES，非通用 |
| OpenTelemetry | ≥ 0.100 | Trace 采集 | CNCF 标准、SDK 覆盖 Go/Python，供应商无关 | Zipkin | Zipkin 不支持 OTLP，生态落后 |
| Jaeger | ≥ 1.57 | Trace 查询 UI | 原生 OTLP 接收、与 OTel Collector 无缝对接 | Tempo | Tempo 需额外 Grafana；Jaeger 独立 UI 更方便开发调试 |
| Vector | ≥ 0.38 | 日志管道 | Rust 实现零拷贝、支持 Kafka Source / Sink、VRL 转换语言 | Logstash / Fluentd | Logstash 内存占用大；Fluentd 性能差 |
| HashiCorp Vault | ≥ 1.16 | 密钥管理 | 动态凭证、租约自动轮转、Transit Engine 加密即服务 | 云 KMS | 私有化部署场景必须离线；公有云可对接 AWS KMS / 阿里云 KMS |
| Istio | ≥ 1.20 | 服务网格 | 零代码 mTLS、灰度路由、Envoy 熔断/重试、可观测注入 | Linkerd / Consul Mesh | Linkerd 功能弱（无 VirtualService）；Consul 与 K8s 绑定过紧，运维复杂 |
