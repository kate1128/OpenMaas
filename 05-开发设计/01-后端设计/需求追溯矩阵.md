- ❌ 阻塞

## 4. 变更影响分析

当 PRD 需求发生变更时，使用此表评估影响范围：

| 变更类型 | 影响评估步骤 |
|---------|------------|
| 新增需求 | 1. 分配 REQ-ID → 2. 识别涉及的设计文档 → 3. 新增测试用例 → 4. 更新此矩阵 |
| 修改需求 | 1. 找到对应 REQ-ID → 2. 评估设计文档是否需要更新 → 3. 评估测试用例是否需要修改 |
| 删除需求 | 1. 标记 REQ-ID 为"已删除" → 2. 检查相关代码是否可清除 → 3. 归档相关测试 |
