## 1. 服务可用性承诺

### 1.1 可用性等级定义

| 服务套餐 | 月可用性承诺 | 年可用性承诺 | 最大月停机时长 |
|---------|-----------|-----------|-------------|
| **基础版** | 99.5% | 99.5% | 3.6 小时/月 |
| **专业版** | 99.9% | 99.9% | 43.8 分钟/月 |
| **企业版** | 99.95% | 99.95% | 21.9 分钟/月 |

### 1.2 可用性计算方式

$$
可用性 = \frac{月总分钟数 - 故障分钟数}{月总分钟数} \times 100\%
$$

**计入停机时间的情况：**
- API 请求错误率 > 5%（持续超过 5 分钟）
- P99 响应时间 > 10 秒（持续超过 5 分钟）
- 任何导致服务完全不可用的事件

**不计入停机时间的情况：**
- 计划内维护窗口（提前 48 小时公告，每月最多 4 小时）
- 厂商 API 自身故障（不可控外部因素）
- 因客户使用超出配额导致的限流
- 不可抗力（自然灾害、政府强制措施等）

## 2. 性能指标承诺

### 2.1 API 响应时间（MaaS 平台自身处理延迟，不含厂商时延）

| 指标 | 基础版 | 专业版 | 企业版 |
|------|--------|--------|--------|
| P50 延迟 | < 20ms | < 10ms | < 5ms |
| P95 延迟 | < 100ms | < 50ms | < 30ms |
| P99 延迟 | < 500ms | < 200ms | < 100ms |

> 注：以上延迟为 MaaS 平台网关至用户的额外延迟，不包含大模型推理时间

### 2.2 其他性能指标

| 指标 | 承诺值 |
|------|--------|
| 语义缓存命中时延（L2 Milvus） | < 50ms |

| 账单查询响应时间 | < 2秒 |
| 精调作业提交响应时间 | < 5秒 |

### 2.3 吞吐量承诺

|------|-------------|-------------|---------|

| 企业版 | 按需协商 | 按需协商 | 按需协商 |

## 3. 故障分级与响应时间

### 3.1 故障等级定义

| 等级 | 定义 | 示例场景 |
|------|------|---------|
| **P0 - 紧急** | 服务完全中断，影响所有用户 | API 全量不可用、认证服务崩溃 |
| **P1 - 严重** | 核心功能受损，影响 > 30% 用户 | 单厂商全量失败无法 Failover |
| **P2 - 重要** | 部分功能受损，影响 < 30% 用户 | 特定模型不可用、监控延迟 |
| **P3 - 一般** | 轻微影响，有临时解决方案 | 账单显示延迟、非核心功能异常 |

### 3.2 响应与解决时间承诺

| 等级 | 响应时间 | 初步评估 | 缓解目标 | 解决目标 |
|------|---------|---------|---------|---------|
| **P0** | 5 分钟 | 15 分钟 | 30 分钟 | 4 小时 |
| **P1** | 15 分钟 | 30 分钟 | 2 小时 | 8 小时 |
| **P2** | 30 分钟 | 2 小时 | 8 小时 | 24 小时 |
| **P3** | 2 小时 | 8 小时 | 48 小时 | 7 个工作日 |

## 4. SLA 赔偿条款

### 4.1 赔偿标准

| 月可用性 | 赔偿比例（对应月费用） |
|---------|-------------------|
| 99.0% ~ 99.5%（基础版未达标） | 10% |
| 98.0% ~ 99.0% | 25% |
| 95.0% ~ 98.0% | 50% |
| < 95.0% | 100% |

| 月可用性（专业版/企业版） | 赔偿比例 |
|------------------------|---------|
| 99.5% ~ 99.9%（专业版未达标） | 10% |
| 99.0% ~ 99.5% | 25% |
| 98.0% ~ 99.0% | 50% |
| < 98.0% | 100% |

### 4.2 赔偿方式

- 赔偿以**账户余额/代金券**形式发放，不支持现金退款
- 赔偿金额最高不超过当月账单总额的 100%
- 赔偿申请须在故障发生后 **30 个自然日内**提交
- 平台在收到申请后 **15 个工作日内**完成审核并发放

### 4.3 赔偿申请流程

```
1. 客户通过控制台或邮件提交赔偿申请
   - 需提供：故障时间段、受影响的 API 请求量证明

2. 平台 SRE 查阅历史监控数据（VictoriaMetrics）验证

3. 平台计算实际可用性
   - 以平台监控数据为准（客户可在控制台查看历史可用性）

4. 结果通知客户，发放代金券
```

## 5. 数据安全承诺

| 承诺项 | 说明 |
|--------|------|
| 数据隔离 | 多租户数据完全隔离，row-level security 强制执行 |
| Prompt 零存储 | API 请求内容（Prompt）不持久化，不用于任何训练 |
| 数据加密 | 传输 TLS 1.3，存储 AES-256 |
| 密钥安全 | 厂商 API Key 通过 KMS 加密，员工无法访问明文 |
| 账单数据留存 | 账单记录保留 7 年，符合财务合规要求 |
| 安全事件通知 | 发现影响客户数据的安全事件，72 小时内通知客户 |

## 6. 服务维护窗口

### 6.1 计划维护规则

- **频率：** 每月最多 2 次，每次最多 4 小时
- **时间：** 周日 02:00 - 06:00（UTC+8）
- **公告：** 提前 **48 小时**通过控制台公告 + 邮件通知
- **紧急维护：** 影响安全的紧急修复，提前 **4 小时**通知

### 6.2 维护期间的服务策略

```
计划维护期间：
- 非流式 API 请求：返回 503，Header: Retry-After: {维护结束时间}
- 已在传输的流式请求：尽量完成，超时后中断
- 账单计算：维护期间的计费暂停，维护结束后补充计算
- 不影响：已提交的精调作业（后台继续运行）
```

## 8. SLA 监控与报告

### 8.1 实时可用性查询

客户可在控制台 **设置 → 服务状态** 页面查看：
- 当前月实时可用性（每 5 分钟更新）
- 历史月度可用性报告（最近 12 个月）
- 各 API 端点历史错误率图表

### 8.2 月度服务报告

- 每月 5 日前，专业版和企业版客户将收到上月服务报告
- 包含：可用性统计、性能指标、故障事件摘要、下月维护计划

## 9. 免责声明

以下情况下，MaaS 平台不承担 SLA 责任：

1. **厂商侧故障：** 第三方大模型厂商（OpenAI/Anthropic/阿里/百度/智谱）自身 API 不可用
2. **客户配置错误：** 由客户侧配置问题导致的请求失败（如无效 API Key、请求格式错误）
3. **超配额限流：** 客户请求量超过购买的配额上限
4. **网络运营商：** 客户侧网络连接问题
5. **不可抗力：** 自然灾害、停电、战争、政府强制措施、大规模 DDoS 攻击超出防护能力
