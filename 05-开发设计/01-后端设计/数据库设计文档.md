## 1. 设计规范

### 1.1 命名规范

| 对象 | 规范 | 示例 |
|------|------|------|
| 数据库 | 小写下划线 | `maas_platform` |
| 表名 | 小写下划线，复数 | `api_keys`, `model_instances` |
| 字段名 | 小写下划线 | `created_at`, `tenant_id` |
| 主键 | `id`，类型 `BIGSERIAL` 或 `UUID` | `id UUID PRIMARY KEY` |
| 外键 | `{关联表单数}_id` | `tenant_id`, `project_id` |
| 时间字段 | `created_at` / `updated_at` / `deleted_at` | TIMESTAMPTZ |
| 软删除 | `deleted_at IS NULL` 标记 | TIMESTAMPTZ 可空 |
| 枚举状态 | `VARCHAR(50)` + CHECK约束 | `status VARCHAR(50)` |

### 1.2 通用字段约定

每张业务主表都包含以下基础字段：

```sql
id          UUID          PRIMARY KEY DEFAULT gen_random_uuid(),
created_at  TIMESTAMPTZ   NOT NULL DEFAULT NOW(),
updated_at  TIMESTAMPTZ   NOT NULL DEFAULT NOW(),
deleted_at  TIMESTAMPTZ   NULL,  -- 软删除标记
created_by  UUID          NULL,  -- 操作人
```

## 3. 核心表结构设计

### 3.1 租户管理（auth schema）

#### 3.1.1 tenants — 租户表

```sql
CREATE TABLE auth.tenants (
    id              UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    name            VARCHAR(200)    NOT NULL,
    slug            VARCHAR(100)    NOT NULL UNIQUE,          -- URL友好标识
    plan_type       VARCHAR(50)     NOT NULL DEFAULT 'free'   -- free/pro/enterprise
                    CHECK (plan_type IN ('free', 'pro', 'enterprise')),
    status          VARCHAR(50)     NOT NULL DEFAULT 'active'
                    CHECK (status IN ('active', 'suspended', 'closed')),
    settings        JSONB           NOT NULL DEFAULT '{}',    -- 租户级配置
    created_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    deleted_at      TIMESTAMPTZ     NULL
);

COMMENT ON TABLE auth.tenants IS '租户（企业/组织）主表';
COMMENT ON COLUMN auth.tenants.slug IS '租户唯一标识，用于子域名等场景';
COMMENT ON COLUMN auth.tenants.settings IS '租户配置JSON，包含内容安全策略、自定义路由等';
```

#### 3.1.2 users — 用户表

```sql
CREATE TABLE auth.users (
```

#### 3.1.3 api_keys — API密钥表（V2.0 扩展 — PRD §01 9.2）

```sql
CREATE TABLE auth.api_keys (
    id              UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id       UUID            NOT NULL REFERENCES auth.tenants(id),
    project_id      UUID            NULL REFERENCES project.projects(id),
    user_id         UUID            NOT NULL REFERENCES auth.users(id),
    name            VARCHAR(200)    NOT NULL,
    key_type        VARCHAR(20)     NOT NULL DEFAULT 'mk-prod'  -- V2.0新增：5种Key类型
                    CHECK (key_type IN ('mk-prod', 'mk-dev', 'mk-ci', 'mk-pg', 'mk-sub')),
    key_hash        VARCHAR(64)     NOT NULL UNIQUE,            -- SHA-256哈希，不存明文
    key_prefix      VARCHAR(20)     NOT NULL,                   -- 展示前缀 mk-prod-xxxx
    scope           TEXT[]          NOT NULL DEFAULT '{chat:completion}', -- V2.0新增：权限范围
    status          VARCHAR(50)     NOT NULL DEFAULT 'active'
                    CHECK (status IN ('active', 'suspended', 'disabled', 'revoked', 'expired')),
    token_quota     BIGINT          NULL,                       -- NULL表示不限制
    token_used      BIGINT          NOT NULL DEFAULT 0,
    rate_limit_rpm  INTEGER         NULL,                       -- 每分钟请求数限制
    rate_limit_tpm  BIGINT          NULL,                       -- 每分钟Token数限制
    ip_whitelist    TEXT[]          NULL,                       -- IP白名单
    zdr_mode        VARCHAR(32)     NOT NULL DEFAULT 'DISABLED' -- V2.0新增：ZDR模式
                    CHECK (zdr_mode IN ('DISABLED', 'METADATA_ONLY', 'TRUE_ZERO_RETENTION')),
    rotation_grace_period INTERVAL  NULL,                       -- V2.0新增：轮转宽限期
    leak_detected_at TIMESTAMPTZ    NULL,                       -- V2.0新增：泄露检测时间
    expires_at      TIMESTAMPTZ     NULL,
    last_used_at    TIMESTAMPTZ     NULL,
    created_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    deleted_at      TIMESTAMPTZ     NULL
);

COMMENT ON TABLE auth.api_keys IS 'API密钥表（V2.0：5种Key类型 + scope + ZDR），密钥明文不入库';
COMMENT ON COLUMN auth.api_keys.key_type IS 'mk-prod=生产 / mk-dev=开发 / mk-ci=CI/CD / mk-pg=Playground / mk-sub=Sub Key';
COMMENT ON COLUMN auth.api_keys.scope IS '权限范围数组，如 {chat:completion, embedding, model:read}';
COMMENT ON COLUMN auth.api_keys.zdr_mode IS '零数据保留模式：DISABLED / METADATA_ONLY / TRUE_ZERO_RETENTION';
```

**V2.0 Key 类型说明（PRD §01 9.2）：**

| 类型 | 前缀 | 典型用法 | 特点 |
|------|------|---------|------|
| **Production** | `mk-prod-` | 线上应用 | 默认，完整功能 |
| **Development** | `mk-dev-` | 本地开发 | Token配额限制更严，可开启debug |
| **CI/CD** | `mk-ci-` | CI流水线 | 特定IP白名单，低成本 |
| **Playground** | `mk-pg-` | Console体验 | 时长限制，低RPM |
| **Sub Key** | `mk-sub-` | 多项目隔离 | 继承父Key配额，不可超父Key |

#### 3.1.4 roles 和 permissions — RBAC权限

```sql
CREATE TABLE auth.roles (
    id              UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id       UUID            NULL REFERENCES auth.tenants(id), -- NULL为全局角色
    name            VARCHAR(100)    NOT NULL,
    description     TEXT            NULL,
    is_system       BOOLEAN         NOT NULL DEFAULT FALSE,  -- 系统内置角色不可删除
    created_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    UNIQUE (tenant_id, name)
);

CREATE TABLE auth.user_roles (
    user_id         UUID            NOT NULL REFERENCES auth.users(id),
    role_id         UUID            NOT NULL REFERENCES auth.roles(id),
    scope_type      VARCHAR(50)     NOT NULL DEFAULT 'tenant'
                    CHECK (scope_type IN ('tenant', 'project', 'global')),
    scope_id        UUID            NULL,                    -- 当scope_type=project时，指向project_id
    granted_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    granted_by      UUID            REFERENCES auth.users(id),
    PRIMARY KEY (user_id, role_id, scope_type, COALESCE(scope_id, '00000000-0000-0000-0000-000000000000'))
);

COMMENT ON TABLE auth.roles IS 'RBAC角色表，支持系统角色和租户自定义角色';
COMMENT ON TABLE auth.user_roles IS '用户角色关联表，支持租户级和项目级授权';
```

### 3.3 模型管理（model schema — V2.0 三层模型架构）

> **V2.0 架构变更**：旧版 `model_registry` 单表平铺改为三层模型架构：`provider_models`（供应商原始模型）→ `vendor_backends`（后端实例，含价格/健康/Key池）→ `logical_models`（面向租户的逻辑模型，50+字段卡片）。详见 `微服务设计/03-model-catalog-service详细设计.md`。

```sql
                    CHECK (rule_type IN ('standard', 'volume', 'reserved', 'promotional')),
    input_price     NUMERIC(12, 8)  NOT NULL,               -- 输入Token单价（元/1000token）
    output_price    NUMERIC(12, 8)  NOT NULL,               -- 输出Token单价（元/1000token）
    min_tokens      BIGINT          NULL,                    -- 阶梯计费起始量
    valid_from      TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    valid_until     TIMESTAMPTZ     NULL,
    created_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW()
);

COMMENT ON TABLE billing.pricing_rules IS '模型计费规则，支持标准价、阶梯价、促销价';
```

### 3.6 审计日志（audit schema）

#### 3.6.1 operation_logs — 操作审计日志

```sql
-- 不可更新、不可删除，Append-Only
CREATE TABLE audit.operation_logs (
    id              BIGSERIAL       PRIMARY KEY,             -- 自增，保障顺序
    tenant_id       UUID            NOT NULL,
    user_id         UUID            NULL,                    -- 系统操作时为NULL
    api_key_id      UUID            NULL,
    request_id      VARCHAR(100)    NULL,
    action          VARCHAR(100)    NOT NULL,                -- 操作类型，如 api_key.create
    resource_type   VARCHAR(100)    NOT NULL,                -- 资源类型
    resource_id     VARCHAR(200)    NULL,                    -- 资源ID
    ip_address      INET            NULL,
    user_agent      TEXT            NULL,
    request_params  JSONB           NULL,                    -- 请求参数（脱敏后）
    response_code   INTEGER         NULL,
    is_success      BOOLEAN         NOT NULL DEFAULT TRUE,
    error_message   TEXT            NULL,
    created_at      TIMESTAMPTZ     NOT NULL DEFAULT NOW()
) PARTITION BY RANGE (created_at);

-- 审计日志表禁止 UPDATE 和 DELETE
CREATE RULE no_update_operation_logs AS ON UPDATE TO audit.operation_logs DO INSTEAD NOTHING;
CREATE RULE no_delete_operation_logs AS ON DELETE TO audit.operation_logs DO INSTEAD NOTHING;

COMMENT ON TABLE audit.operation_logs IS '操作审计日志，Append-Only，不可修改删除';
```
