## 1. 快速开始

### 1.1 前置条件

- 已在 MaaS 平台注册账号
- 已在控制台创建 API Key（[控制台入口](https://console.maas-platform.com)）

### 1.2 Hello World（30秒）

```bash
# 替换 YOUR_API_KEY 为你的 API Key
curl https://api.maas-platform.com/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "qwen-turbo",
    "messages": [
      {"role": "user", "content": "你好，介绍一下你自己"}
```

预期响应：

```json
  "model": "qwen-turbo",
  "choices": [{
    "message": {
      "role": "assistant",
      "content": "你好！我是通义千问..."

  }],
  "usage": {
    "prompt_tokens": 12,
    "completion_tokens": 45,

  }

```

### 1.3 兼容性说明

MaaS 平台完全兼容 **OpenAI API 格式**，只需修改 `base_url` 即可迁移现有代码：

```python
# 迁移前（直接用 OpenAI）

client = openai.OpenAI(
    api_key="YOUR_MAAS_API_KEY",
    base_url="https://api.maas-platform.com/v1"
)
```

## 2. 认证与 API Key

### 2.1 获取 API Key

1. 登录 [控制台](https://console.maas-platform.com)
2. 进入「API Keys」→ 点击「新建 API Key」
3. 填写名称、设置可选限额/IP白名单
4. **复制并妥善保存明文 Key**（仅显示一次）

### 2.2 请求认证

所有 API 请求需在 Header 中携带：

```
Authorization: Bearer YOUR_API_KEY
```

### 2.3 API Key 安全规范

```
✅ 正确做法：
  - 将 API Key 存储在环境变量或密钥管理系统中
  - 不同环境（dev/staging/prod）使用不同的 API Key
  - 为 API Key 设置最小必要的 IP 白名单

❌ 错误做法：
  - 硬编码在源代码中（会被提交到 Git！）
  - 在前端 JavaScript 中使用（会暴露给用户）
  - 多个应用共用一个 Key（无法区分用量）
```

```python
# ✅ 正确：从环境变量读取
import os
api_key = os.environ.get("MAAS_API_KEY")

# ❌ 错误：硬编码
api_key = "sk-AbCd..."  # 不要这样做！
```

## 3. 调用大模型 API

### 3.1 基础对话（Chat Completions）

```
POST https://api.maas-platform.com/v1/chat/completions
```

**请求体参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `model` | string | 是 | 模型名称（见模型列表） |
| `messages` | array | 是 | 对话消息数组 |
| `max_tokens` | integer | 否 | 最大输出 Token 数，默认 4096 |
| `temperature` | float | 否 | 随机性 0-2，默认 1.0 |
| `top_p` | float | 否 | 核采样，默认 1.0 |
| `stream` | boolean | 否 | 是否流式输出，默认 false |
| `stop` | string/array | 否 | 停止词 |
| `user` | string | 否 | 最终用户标识（用于审计） |

**MaaS 扩展参数（可选）：**

| 参数 | 类型 | 说明 |
|------|------|------|
| `x-maas-fallback-models` | string | 逗号分隔的备选模型，主模型失败时使用 |
| `x-maas-cache-enabled` | boolean | 是否启用语义缓存，默认 true |
| `x-maas-priority` | string | `realtime`（低延迟）/ `economy`（低成本） |

### 3.2 Embedding 向量化

```
POST https://api.maas-platform.com/v1/embeddings
```

```json

  "model": "text-embedding-3-small",
  "input": "这是需要向量化的文本"
```

## 4. 流式输出

流式输出使用 SSE（Server-Sent Events）格式，适合需要实时渲染的场景（如聊天机器人）。

### 4.1 请求示例

```bash
curl https://api.maas-platform.com/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "gpt-4o",
    "messages": [{"role": "user", "content": "写一首关于春天的诗"}],
    "stream": true
  }'
```

### 4.2 响应格式

每条 SSE 数据格式为 `data: {json}` ，最后一条为 `data: [DONE]`：

```
data: {"id":"chatcmpl-xxx","choices":[{"delta":{"role":"assistant"},"index":0}]}
data: {"id":"chatcmpl-xxx","choices":[{"delta":{"content":"春"},"index":0}]}
data: {"id":"chatcmpl-xxx","choices":[{"delta":{"content":"风"},"index":0}]}
...
data: {"id":"chatcmpl-xxx","choices":[{"delta":{},"finish_reason":"stop","index":0}]}
data: [DONE]
```

## 5. 多语言 SDK 示例

### 5.1 Python

```python
from openai import OpenAI

client = OpenAI(
    api_key="YOUR_MAAS_API_KEY",
    base_url="https://api.maas-platform.com/v1"
)

# 普通调用
response = client.chat.completions.create(
    model="qwen-turbo",
    messages=[
        {"role": "system", "content": "你是一个专业的客服助手"},
        {"role": "user", "content": "如何申请退款？"}
    ],
    temperature=0.7,
    max_tokens=500
)
print(response.choices[0].message.content)

# 流式调用
stream = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "讲个故事"}],
    stream=True
)
for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="", flush=True)
```

### 5.2 Go

```go

import (
    "context"
    "fmt"
    "github.com/sashabaranov/go-openai"

func main() {
    config := openai.DefaultConfig("YOUR_MAAS_API_KEY")
    config.BaseURL = "https://api.maas-platform.com/v1"

    resp, err := client.CreateChatCompletion(

                {Role: "user", Content: "你好"},
            },
        },
    )
    if err != nil {
        panic(err)
    }
    fmt.Println(resp.Choices[0].Message.Content)
}
```

### 5.3 JavaScript / TypeScript

```typescript
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.MAAS_API_KEY,
  baseURL: "https://api.maas-platform.com/v1",
});

// 异步调用
async function chat(userMessage: string): Promise<string> {
  const response = await client.chat.completions.create({
    model: "gpt-4o",
    messages: [{ role: "user", content: userMessage }],
    max_tokens: 1000,
  });
  return response.choices[0].message.content ?? "";
}

// 流式调用（适合 Next.js API Route）
async function streamChat(userMessage: string) {
  const stream = await client.chat.completions.create({
    model: "gpt-4o",
    messages: [{ role: "user", content: userMessage }],
    stream: true,
  });

  for await (const chunk of stream) {
    const content = chunk.choices[0]?.delta?.content ?? "";
    process.stdout.write(content);
  }
}
```

## 7. 错误处理与重试

### 7.1 错误码说明

| HTTP 状态码 | 错误码 | 说明 | 处理建议 |
|------------|--------|------|---------|
| 400 | `invalid_request` | 请求参数错误 | 检查请求格式 |
| 401 | `unauthorized` | API Key 无效或已失效 | 检查 API Key |
| 403 | `forbidden` | 无权访问该模型或功能 | 检查权限配置 |
| 429 | `rate_limit_exceeded` | 超过速率限制 | 降低请求频率 |
| 429 | `quota_exceeded` | Token 配额耗尽 | 充值或申请提额 |
| 503 | `model_unavailable` | 模型服务暂时不可用 | 重试或切换备用模型 |
| 504 | `timeout` | 请求超时 | 降低 max_tokens 或重试 |

### 7.2 带指数退避的重试实现

```python
import time
import random
from openai import OpenAI, RateLimitError, APIStatusError

client = OpenAI(
    api_key="YOUR_MAAS_API_KEY",
    base_url="https://api.maas-platform.com/v1"
)

def chat_with_retry(messages, model="qwen-turbo", max_retries=3):
    """带指数退避重试的对话函数"""
    retryable_status = {429, 503, 504}

    for attempt in range(max_retries):
        try:
            return client.chat.completions.create(
                model=model,
                messages=messages
            )
        except RateLimitError:
            if attempt == max_retries - 1:
                raise
            wait = (2 ** attempt) + random.uniform(0, 1)
            print(f"Rate limited, retrying in {wait:.1f}s...")
            time.sleep(wait)
        except APIStatusError as e:
            if e.status_code in retryable_status and attempt < max_retries - 1:
                wait = (2 ** attempt) + random.uniform(0, 1)
                print(f"API error {e.status_code}, retrying in {wait:.1f}s...")
                time.sleep(wait)
            else:
                raise

# 使用示例
response = chat_with_retry(
    messages=[{"role": "user", "content": "你好"}],
    model="qwen-turbo"
)
```

### 7.3 多模型 Fallback

```python
FALLBACK_MODELS = ["gpt-4o", "claude-3-5-haiku", "qwen-plus"]

def chat_with_fallback(messages):
    """主模型失败时自动切换备用模型"""
    for model in FALLBACK_MODELS:
        try:
            response = client.chat.completions.create(
                model=model,
                messages=messages,
                timeout=10
            )
            return response
        except Exception as e:
            print(f"Model {model} failed: {e}, trying next...")
            continue
    raise Exception("All fallback models exhausted")
```

## 8. 速率限制与配额

### 8.1 限制说明

| 限制类型 | 默认值 | 说明 |
|---------|--------|------|
| RPM（每分钟请求数） | 1,000 | 可按需申请提升 |

## 10. 最佳实践与成本优化

### 10.1 Prompt 优化

```
✅ 减少 Token 消耗的技巧：

1. 精简 System Prompt
   - 避免每次请求都附带长篇 System Prompt（500+ tokens）
   - 提炼核心指令，去除冗余描述

2. 合理设置 max_tokens
   - 根据预期输出长度设置，不要无限大
   - 短对话场景设 200-500 即可

3. 利用语义缓存
   - 相似问题会命中缓存，不消耗 Token
   - 保持 Prompt 模板一致性可提高缓存命中率

4. 选择合适的模型
   - 简单问答用 qwen-turbo（比 gpt-4o 便宜约 10 倍）
   - 仅复杂任务使用旗舰模型
```

### 10.2 高并发场景

```python
# 使用 asyncio 并发调用（适合批量处理）
import asyncio
from openai import AsyncOpenAI

async_client = AsyncOpenAI(
    api_key="YOUR_MAAS_API_KEY",
    base_url="https://api.maas-platform.com/v1"
)

async def process_batch(questions: list[str]) -> list[str]:
    tasks = [
        async_client.chat.completions.create(
            model="qwen-turbo",
            messages=[{"role": "user", "content": q}],
            max_tokens=200
        )
        for q in questions
    ]
    responses = await asyncio.gather(*tasks, return_exceptions=True)
    return [
        r.choices[0].message.content if not isinstance(r, Exception) else f"Error: {r}"
        for r in responses
    ]

# 并发处理 10 个问题
questions = ["问题1", "问题2", ..., "问题10"]
answers = asyncio.run(process_batch(questions))
```

### 10.3 成本控制建议

| 场景 | 建议 | 预期节省 |
|------|------|---------|
| 重复性问答（FAQ） | 开启语义缓存（默认开启） | 30-50% |
| 多轮对话 | 只保留近 N 轮上下文，不要传入全部历史 | 20-40% |
| 文档处理 | 先用向量搜索找到相关段落，再调用 LLM | 50-70% |
| 调试阶段 | 为 dev API Key 设置月度 Token 上限 | 防止超支 |
| 高峰批量任务 | 使用 `economy` 优先级，允许用更便宜的模型 | 40-60% |
