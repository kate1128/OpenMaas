## 1. 平台简介

MaaS（Model as a Service）是一个**大模型聚合调度平台**，帮助企业开发者：

- 用一个 API Key 接入 OpenAI、Anthropic、通义千问、文心一言等主流大模型
- 通过智能路由自动选择性价比最优的模型
- 统一监控用量、控制成本、管理权限
- 一站式完成模型精调和私有化部署

**核心价值：**

```
无需多账号管理 → 统一 API 格式 → 智能降本 → 数据安全合规
```

## 2. 注册与登录

### 2.1 账号注册

1. 访问 [https://console.maas-platform.com](https://console.maas-platform.com)
2. 点击「申请开通」，填写企业信息和联系方式
3. 审核通过后（1-2 个工作日）收到开通邮件
4. 按邮件指引完成密码设置

> **企业 SSO 登录：** 如果你的企业已配置 SSO（钉钉/企业微信/Okta），可直接使用企业账号登录，无需单独注册。联系企业管理员开通 SSO 集成。

### 2.2 多租户说明

```
企业账号（租户）
  └── 项目 A（如：生产环境）
  │     └── API Key 1、API Key 2...
  └── 项目 B（如：测试环境）
        └── API Key 3...
```

- **租户**：代表一个企业或独立业务单元，独立计费
- **项目**：租户下的业务分组，用于隔离不同环境或业务线
- **API Key**：绑定到具体项目，代表一个调用凭证

### 2.3 角色权限说明

| 角色 | 可以做什么 |
|------|----------|
| 开发者 | 创建/管理自己的 API Key，查看本项目用量 |
| 项目管理员 | 管理项目成员，配置路由策略，查看项目账单 |
| 平台管理员 | 管理所有租户，配置全局模型，查看全平台数据 |
| 审计员 | 只读访问账单和审计日志 |

## 4. API Key 管理

### 4.1 创建 API Key

1. 点击左侧「🔑 API Keys」
2. 点击右上角「新建 API Key」
3. 填写以下信息：
   - **名称**：便于识别的名称（如 `prod-backend`）
   - **Token 额度**：月度 Token 上限（留空表示不限制）
   - **速率限制**：每分钟最大请求数
   - **IP 白名单**：仅允许指定 IP 调用（可选，推荐生产环境配置）
4. 点击「创建」，**立即复制并保存明文 Key**

> ⚠️ **重要：** API Key 明文仅在创建时显示一次，关闭弹窗后无法再次查看。请立即保存到密码管理工具或环境变量。

### 4.2 查看 Key 用量

在 API Key 列表中，每个 Key 显示：
- 今日调用次数
- 额度余量（已用 / 总额度）
- 最后使用时间

点击某个 Key 的「查看日志」可以看到近期调用详情。

### 4.3 禁用 / 吊销 Key

- **禁用**：临时停止该 Key 的使用（可恢复）
- **吊销**：永久删除（不可恢复，需谨慎）

Key 被吊销后立即失效，使用该 Key 的调用会立即返回 401 错误。

## 5. 模型广场

### 5.1 浏览模型

模型广场展示平台上所有可用的大模型，支持：
- **按类型筛选**：对话 / 代码 / 嵌入 / 多模态
- **按厂商筛选**：OpenAI / Anthropic / 阿里云 / 百度...
- **按能力标签**：中文优化 / 长上下文 / 推理增强...

### 5.2 模型卡片信息

每个模型卡片显示：
- **可用状态**：🟢 可用 / 🟡 降级（高延迟）/ 🔴 不可用

- **平均延迟**：P50 / P95 延迟参考

### 5.3 查看模型详情

点击模型卡片的「查看详情」可查看：
- 详细能力描述和适用场景
- 上下文窗口大小
- 历史性能趋势图
- 使用限制说明

点击「在 Playground 试用」直接跳转到调试页。

## 6. Playground 在线调试

Playground 允许你在控制台直接调试模型，无需写代码。

### 6.1 基础使用

1. 进入「🧪 Playground」
2. 从下拉列表选择要调试的模型
3. 在「System」框中输入系统提示词（可选）
4. 在下方输入框输入用户消息，点击「发送」
5. 右侧面板显示模型回复和 Token 消耗

### 6.2 参数调节

Playground 右侧面板支持调节：

| 参数 | 说明 | 推荐范围 |
|------|------|---------|
| Temperature | 输出随机性，越高越发散 | 0.7（对话）/ 0（代码） |
| Max Tokens | 最大输出长度 | 按需设置，避免过大 |
| Top P | 核采样，与 Temperature 配合 | 通常保持默认 1.0 |
| Stop Sequences | 遇到该词停止输出 | 按需配置 |

### 6.3 保存与分享

- 点击「保存对话」可将当前对话保存为草稿
- 点击「生成代码」可自动生成对应的 API 调用代码（支持 Python/JS/Go）
- 点击「分享」生成可访问的只读链接，方便与团队分享调试结果

> ⚠️ **费用提示：** Playground 的每次调用都会计入你的项目用量。建议调试完成后关闭不必要的会话。

## 7. 用量统计与账单

### 7.1 用量统计

进入「📈 用量账单」→「用量统计」：

**筛选维度：**
- 时间范围：自定义 / 今日 / 本周 / 本月
- 分组方式：按模型 / 按 API Key / 按天
- 数据粒度：小时 / 天 / 周

**图表类型：**
- Token 用量趋势（折线图）
- 模型分布（饼图）
- 成本趋势（柱状图）

### 7.2 账单查询

进入「用量账单」→「账单」：

- **本月账单**：实时估算当前月累计费用
- **历史账单**：按月查看已出账的详细账单
- **账单导出**：下载 CSV 格式账单，方便财务报销

**账单出账时间：** 每月1日生成上个自然月的账单，3个工作日内完成对账。

### 7.3 预算管理

在「项目设置 → 预算」中可配置：

- **月度预算上限**：超出后暂停该项目的 API 调用（并提前告警）
- **告警阈值**：用量达到预算的 X% 时发送通知（如 70%、90%）
- **通知方式**：邮件 / 钉钉 / 控制台通知

## 8. 路由策略配置

路由策略决定你的 API 请求会被发到哪个模型以及按什么顺序 Fallback。

### 8.1 理解默认策略

新创建的项目默认使用**综合路由策略**：自动在成本、延迟、质量之间取得平衡。大多数场景下无需修改。

### 8.2 创建自定义策略

适合以下场景：
- 希望固定使用某个特定模型
- 有严格的成本控制需求（如只用便宜模型）
- 特殊合规要求（如数据不出境，只用国产模型）

**配置步骤：**
1. 进入「🔀 路由策略」→「新建策略」
2. 调整三个权重滑块（成本 / 延迟 / 成功率）
3. 从模型列表添加候选模型并设置权重
4. 配置 Failover 顺序（主模型不可用时的备选链）
5. 点击「保存并生效」

### 8.3 智能建议

页面顶部的 Copilot 建议栏会根据你的实际调用分析，主动给出路由优化建议，如：
> "检测到 73% 的请求为简单问答，切换到经济模式预计节省 ¥120/月"

## 9. 精调中心

精调（Fine-tuning）让你基于自己的数据训练出专属模型，提升特定场景下的准确率。

### 9.1 准备训练数据

数据格式要求（JSONL 格式，UTF-8 编码）：

```jsonl
{"messages": [{"role": "system", "content": "你是专业客服"}, {"role": "user", "content": "如何退款"}, {"role": "assistant", "content": "您可以在订单页面..."}]}
{"messages": [{"role": "user", "content": "营业时间是几点"}, {"role": "assistant", "content": "我们的营业时间是..."}]}
```

**数据量建议：**
- LoRA 精调：至少 100 条，建议 500-2000 条
- 全量精调（SFT）：建议 2000 条以上

### 9.2 创建精调任务

1. 进入「⚗️ 精调中心」→「新建精调任务」
2. **Step 1 基础配置：** 填写任务名、选择基础模型（如 Qwen-7B-Chat）、选择精调方式（LoRA 推荐）
3. **Step 2 上传数据集：** 上传训练集（必填）和验证集（可选），支持上传到平台数据集库
4. **Step 3 超参数：** 通常保持默认即可。高级用户可调节学习率、训练轮数等
5. **Step 4 确认：** 查看预估费用和 GPU 时长，点击「提交任务」

### 9.3 查看任务进度

精调任务列表显示：
- **状态**：排队中 / 训练中 / 已完成 / 失败
- **进度条**：当前训练轮次进度
- **GPU 用量**：已消耗 GPU 小时数
- **训练日志**：实时查看 loss 等训练指标

### 9.4 部署并使用精调模型

任务完成后：
1. 点击「部署」，平台将精调模型部署为推理端点（约5-10分钟）
2. 部署后可在模型广场「我的模型」中看到该模型
3. 使用 API 调用时，`model` 参数填写精调模型 ID（如 `ft:qwen-7b:job_xxx`）

## 10. 监控告警中心

### 10.1 查看实时监控

进入「📡 监控告警中心」：

- **系统状态总览**：各服务和模型端点的健康状态（绿/黄/红）
- **实时指标图表**：QPS、成功率、延迟趋势（最近1小时 / 24小时）
- **活跃告警列表**：当前未处理的告警

### 10.2 告警处理

每条告警显示：
- 告警级别（P0-P3）
- 触发时间和持续时长
- 告警内容（当前值 vs 阈值）

操作按钮：
- **查看详情**：跳转到对应监控图表
- **静默**：在指定时间内不再触发该告警（维护窗口时使用）
- **已处理**：标记为已处理

### 10.3 配置自定义告警

在「配置告警规则」中可为你的项目添加自定义告警：
- 错误率超过某个阈值
- 月度费用超过预算百分比
- 特定 API Key 的异常活动

## 12. MaaS Copilot 使用指南

### 12.1 打开 Copilot

点击页面右下角的「💬 Copilot」悬浮按钮，弹出对话窗口。

### 12.2 Copilot 能帮你做什么

**集成帮助：**
- "帮我写 Python 调用示例"
- "如何在 Next.js 中接入流式输出"
- "用 Go 实现带重试的封装"

**问题诊断：**
- "为什么我的请求报 429 错误"
- "最近错误率上升是什么原因"
- "这条日志是什么意思"（可直接粘贴错误信息）

**成本分析：**
- "本月为什么费用这么高"
- "哪个 API Key 用量最多"
- "怎么降低我的 Token 消耗"

**模型选择：**
- "我要做一个中文客服机器人，推荐什么模型"
- "有没有比 GPT-4o 便宜但效果差不多的模型"

### 12.3 使用技巧

- **附上错误信息**：直接粘贴错误日志，Copilot 能更准确地诊断
- **说明业务场景**：越具体的描述，推荐越精准
- **点击操作按钮**：Copilot 的回复中会附带「查看详情」「一键执行」等快捷按钮，无需手动跳转

## 13. 常见问题（FAQ）

**Q：API Key 不小心泄露了怎么办？**  
A：立即前往控制台「API Keys」页面，找到该 Key 点击「吊销」。吊销后立即生效，泄露的 Key 将无法再使用。然后创建一个新的 Key 替换。

**Q：为什么我的请求返回 503？**  
A：503 表示模型服务暂时不可用，通常几分钟内会自动恢复。你可以：
1. 稍后重试（建议使用指数退避重试逻辑）
2. 在路由策略中配置备用模型，平台会自动 Failover

**Q：Token 消耗完了怎么办？**  
A：前往「用量账单」→「账单」页面充值，或联系管理员提升配额。如果是月度额度耗尽，等下个月重置或临时调整上限。

**Q：能不能只用国产模型（数据不出境）？**  
A：可以。在「路由策略」中创建自定义策略，只添加通义千问、文心一言、GLM 等国产模型，取消其他模型即可。

**Q：精调任务排队等待太久怎么办？**  
A：GPU 资源是动态分配的，高峰期会有排队。建议：
- 选择非高峰时段提交（凌晨/周末）
- 联系管理员申请优先资源配额

**Q：如何查看 API 的完整请求/响应日志？**  
A：在「API Keys」→ 点击某个 Key 的「查看日志」，可以看到最近7天的调用明细。出于隐私保护，请求内容默认不记录，如需开启可在项目设置中配置。

**Q：Playground 的对话记录会保存多久？**  
A：Playground 对话记录保存7天。如需长期保存，请使用「保存对话」功能手动保存。
